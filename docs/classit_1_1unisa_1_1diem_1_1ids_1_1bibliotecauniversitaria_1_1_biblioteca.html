◆ registraPrestito()

LocalDate it.unisa.diem.ids.bibliotecauniversitaria.Biblioteca.registraPrestito ( Libro libro,
Utente utente,
LocalDate dataInizio,
LocalDate dataRestitPrevista )
inline

Registra un nuovo prestito nel sistema per un libro e un utente specifici, dopo aver effettuato i controlli di validità.

Questo metodo implementa il flusso principale del caso d'uso "Registrazione Prestito",

Parameters
libroL'oggetto Libro che viene prestato.
utenteL'oggetto Utente a cui viene effettuato il prestito.
dataInizioLa data effettiva di inizio del prestito.
dataRestitPrevistaLa data prevista per la restituzione (IF-6.2).
Returns
La data di inizio prestito, come conferma dell'avvenuta registrazione.
Precondition
L'utente e il libro devono essere già registrati nell'archivio.
Il libro deve avere copie disponibili (IF-6.4).
L'utente non deve avere più di 3 prestiti contemporaneamente (IF-6.5).
L'utente non deve avere consegne in ritardo (IF-6.6).
L'utente non deve essere sospeso (IF-6.7).
Il personale ha effettuato l'accesso al sistema e visualizza il menu (UI-4).
Postcondition
Il prestito è registrato con data di inizio e di restituzione prevista.
Il numero di copie disponibili del libro è decrementato.
Il numero di prestiti dell'utente è incrementato.

◆ rimuoviUtente()

void it.unisa.diem.ids.bibliotecauniversitaria.Biblioteca.rimuoviUtente ( String matricola)
inline

Rimuove un utente specifico dal sistema tramite la sua matricola.

Questo metodo implementa il flusso principale del caso d'uso "Rimozione Utenti". Prima della rimozione, il sistema esegue un controllo per verificare la presenza di prestiti attivi (IF-9.4). Se non ci sono prestiti attivi, i dati dell'utente (DF-2) vengono rimossi dall'archivio (IF-1).

Parameters
matricolaLa matricola (identificativo univoco) dell'utente da rimuovere.
Precondition
Il personale ha effettuato l'accesso al sistema.
Il personale visualizza l'interfaccia "Menu" (UI-4).
Postcondition
Un utente viene rimosso dal sistema.
